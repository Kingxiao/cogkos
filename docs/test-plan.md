## 测试执行策略

### 单元测试 (70%)
- 每个模块独立验证
- Mock 外部依赖
- 并行执行

### 集成测试 (30%)
- 需要 Docker 环境 (PG/FalkorDB/Qdrant/MinIO)
- 按依赖顺序执行
- 失败时保留日志

### 执行顺序

```
1. L1 持久化层 (基础依赖)
2. L4 语义索引层 + L5 知识图谱层 (存储层)
3. L6 MCP 查询层 (核心逻辑)
4. L9 摄入管道层 (写入路径)
5. L7 外部知识层 + L8 联邦层 (扩展功能)
6. L2 进化引擎 + L3 异步整合 (后台任务)
7. 集成测试 (E2E)
```
