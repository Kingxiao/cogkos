# CogKOS 实施路线图

## 总体节奏

```
Phase 1 ──→ Phase 2 ──→ Phase 3 ──→ Phase 4 ──→ Phase 5
 基建        闭环        深化        联邦        自反
(3个月)     (3个月)     (3个月)     (3个月)     (持续)
```

> **Phase vs V 的关系**：Phase 1-5 是**功能迭代**（做什么），V1-V3 是**规模版本**（撑多少 Agent）。两者正交——Phase 1 可以在 V1 上跑，也可以在 V2 上跑。详见 [ARCHITECTURE.md 规模化路径](./ARCHITECTURE.md#规模化路径从-10k-到-10m-agent)。

## 技术栈

| 组件 | 选型 |
|------|------|
| 后端 | **Rust** |
| 图数据库 | **FalkorDB** |
| 向量库 | **PostgreSQL pgvector** (BGE-M3 1024d / text-embedding-3-large 3072d) |
| 关系库 | **PostgreSQL** |
| 对象存储 | **S3 兼容** |
| MCP Server | **Rust + rmcp SDK** |
| LLM | **API 调用**（不自训练） |
| 包管理 | **cargo** |

### LLM 成本假设

| Phase | 用途 | 敏感度 |
|-------|------|--------|
| 1 | 仅查询时轻量预测 | 低 |
| 2-3 | Sleep-time 聚合/冲突分析 | 中 |
| 4-5 | 大规模 Insight/范式转换 | 高→随成本下降降低 |
